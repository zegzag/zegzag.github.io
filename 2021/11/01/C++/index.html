C++ 移动语义与右值

C++ 程序里面的变量可以归为三类：

  • 左值(lvalue):
    • 通俗的讲，带有变量名称的值。
  • 纯右值(prvalue):
    • 形如：1+3的表达式的值
    • 形如：1, 'c', true不跟对象关联的值
    • 形如：非引用返回的临时变量值。
      • 由于函数的返回值(非引用类型)的临时变量是右值，移动语义实现时，我们可以通过构造函数进行传参（std中各种emplace的实现）。
  • 将亡值(xvalue)，和右值引用相关:
    • 返回值类型为 T&& 的值
    • std::move()的值
    • 转换为 T&& 类型转换函数的返回值。

C++ 多线程

  稍微复杂一些的环境和模型几乎无法绕开并发编程。典型的场景有

  • 消息队列模型 (生产者消费者模型)
  • GUI编程。
  • 搜索引擎。一个引擎的整个处理过程，从分词，召回，粗排，精排，重排，每一步中各个模块逻辑上能并行处理的 (比如待排doc的特征assign)，异步处理的(比如各特征来源的获取，如果一个特征请求时间很长，就可以在逻辑上在一开始处执行，最后处返回结果，中间的时间多核CPU可以同时来处理其他的逻辑)。如果单以单线程运行的话，那么整个链路的耗时将是难以忍受之长。

  此文对多线程的并发和异步编程做一些归纳总结   一个程序的运行，总体上是由 数据(data) 以及 对数据的操作(func) 组成的。操作是需要 一定的时间duration 执行完成的。操作可以被拆分成多个简单的子操作的（最简单的子操作即CPU指令，编译过程就是把复杂操作拆分成单个CPU指令的过程）。单线程程序即：对同一块数据同一时刻timestamp仅有一个操作进行，多线程即有多个操作进行。要保证操作之间的同步协调，即需要对操作进行原子化。

  • 原子化操作：
    • 最基础的操作就是CPU指令，它本身就是原子化的。
    • 针对data的最基础的两个操作： 读操作(read) 和 写操作(write)，可以被语言设计打包成原子性的操作，这就是 C++ <atomic> 库的内容。《C++ Concurrency in Action》第5, 7章
    • 对于更复杂的用户自定义的各种操作(function)，较为简单的原子化的手段是 锁机制(<mutex>) 头文件的内容，涉及到mutex, lock_guard, unique_lock, conditional_variable等概念。 其本质是 通过锁机制来让线程可以挂起（等待）从而实现对同一数据操作的原子化 的目的。《C++ Concurrency in Action》第3,4,6章
    • 通过设计原子化的数据（即对该数据基础的读操作和写操作进行原子化），保证用户自定义函数内部每一步操作的原子性和同步性，我们可以绕过锁机制来设计concurrent funcion，从而实现更高效的并发。《C++ Concurrency in Action》第7, 8章
